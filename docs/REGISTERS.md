# Registers — How the Two-Stage Pipeline Lets You Choose the Voice

Companion to [`CONSUMER-REPORT.md`](CONSUMER-REPORT.md). Where the Consumer Report uses a single formal-analytical voice, this document catalogues the **other** registers Gemma 4 26B-A4B will produce on demand — same model, same transcript, controlled entirely by the prompt.

A **register** is the dialect / tone / style fitted to a context. Switching registers is what lets you take a transcript meant for an engineering test log and turn it into a press release, a children's book, or a news headline — without re-recording, re-transcribing, or fine-tuning a separate model.

The example transcript used throughout this document is Simon Willison's voice memo (`demo-audio-for-gemma.wav`):

> *"This right here is a quick voice memo. I want to try it out with MLXVLM. Just going to see if it can be transcribed by Gemma and how well that works."*

Each register below was generated by running the same Gemma 4 26B-A4B (UD-MLX-4bit) call against this transcript, varying only the instruction in the prompt.

## Dimensions of register variation

When you write a prompt instruction, you can vary along several independent axes. Most outputs are a combination of choices along each axis.

| Dimension | Pole A | Pole B | Example axis label |
|---|---|---|---|
| **Formality** | Formal | Casual | "as a SEC filing" ↔ "as a group chat message" |
| **Technicality** | Layperson | Specialist | "for a 5th-grader" ↔ "for a postdoc" |
| **Voice** | Third-person objective | First-person subjective | "describe what was said" ↔ "speak as the speaker" |
| **Function** | Informative | Persuasive · Entertaining | "explain" ↔ "convince me" ↔ "make me laugh" |
| **Form** | Prose | Verse · Structured | "paragraph" ↔ "haiku" ↔ "JSON" ↔ "bullet list" |
| **Audience** | General public | Targeted | "for a tabloid reader" ↔ "for a compliance officer" |
| **Tone** | Neutral | Loaded | "neutral" ↔ "skeptical" ↔ "enthusiastic" ↔ "deadpan" |
| **Length** | Compressed | Expansive | "one sentence" ↔ "an essay" |
| **Persona** | Default Gemma | Specific persona | "as a 1920s radio announcer" ↔ "as a snarky teen" |

Any combination is reachable. Examples that didn't make the eight-register demo above (but Gemma will produce on request):

- **BLUF executive brief**: top-loaded, decision-grade ("Bottom line: speaker initiated MLXVLM/Gemma transcription test. Status: in-flight.")
- **Twitter / X thread**: numbered tweets with platform conventions
- **Voiceover script**: read-aloud cadence with stage directions
- **Compliance memo**: passive, attributable, date-anchored
- **Children's book**: simpler vocabulary, concrete imagery, story arc
- **Sardonic / comedic**: dry, deadpan, ironic
- **Forensic / deposition style**: numbered, neutral, evidentiary
- **Marketing / promotional**: active verbs, benefit-framing, calls to action

## When to use each

| Use case | Register | Why |
|---|---|---|
| Internal docs / specs / ADRs (PLAN.md, CONSUMER-REPORT.md) | Formal-analytical | Precision, evidence-binding, neutral voice |
| Broadcast / podcast / radio voiceover | Voiceover script + casual | Read-aloud cadence, contractions OK |
| Onboarding / new-user guide | Plain-language | Reading-grade matched to broadest audience |
| Engineering test logs | Technical-engineering | Greppable, label-prefixed, no ornament |
| Press release / news article | Journalistic | Inverted pyramid; lede-first |
| Story / podcast intro / cold-open | Literary-narrative | Sensory framing, hook |
| Legal / regulatory filings | Bureaucratic-procedural | Passive, formal, defensible |
| Creative repurposing / social hook | Poetic / haiku / aphorism | Compression for shareability |
| Executive summary / status update | BLUF | Decision-grade, top-loaded |
| Social media engagement | Twitter thread / casual | Platform conventions; brevity |
| Children's content / accessibility | Children's book / plain | Simpler vocabulary; concrete imagery |
| Compliance / audit | Forensic / deposition | Date-anchored, neutral, attributable |
| Marketing / promotional | Promotional | Active verbs, benefit-framing |

## Prompt recipes — how to invoke each register

The same `tools/two-stage.sh` invocation handles every register. The only difference is one phrase in the prompt:

```bash
# Formal-analytical (default of CONSUMER-REPORT.md):
./tools/two-stage.sh audio.wav "Summarise in formal analytical register."

# Casual-conversational:
./tools/two-stage.sh audio.wav "Describe in casual conversational English with emoji, as if texting a friend."

# Technical-engineering (test-log):
./tools/two-stage.sh audio.wav "Render as a brief test-log entry with labelled fields (Test Case / Objective / Input / Result)."

# Journalistic:
./tools/two-stage.sh audio.wav "Write a one-sentence news headline plus a one-sentence lede."

# Literary-narrative:
./tools/two-stage.sh audio.wav "Open a short literary story whose first paragraph captures this moment."

# Bureaucratic-procedural:
./tools/two-stage.sh audio.wav "Render in formal bureaucratic register with passive voice and procedural framing."

# Plain-language:
./tools/two-stage.sh audio.wav "Explain in plain language at a fifth-grade reading level."

# Poetic / haiku:
./tools/two-stage.sh audio.wav "Compose one haiku capturing the moment."
```

No fine-tuning, no extra installs, no second model. **Register is a prompt parameter** — you control it on a per-call basis.

## Why this matters

The two-stage pipeline (whisperX → Gemma 4 26B-A4B) is not just a transcription tool. It's a transcription **plus a register switch** in one pass. Audio in, *any* register out. The same recorded conversation can become:

- A formal meeting minutes document for the file
- A casual update for the team Slack
- A compliance memo for legal
- An open-floor narrative for the company blog
- A one-line headline for the press release

…all from the same source audio, all in one run, all for $0 in API cost.
